# 留言板 —— 練「多人共用同一份資料」

**線上網址**：https://guestbook-demo.onrender.com（雲端版接的是 Supabase Postgres，不是本機這個 SQLite 檔）

## 這是什麼

跟記帳本是同一套架構（瀏覽器→伺服器→資料庫），但練的重點不一樣：
記帳本通常只有你自己在看自己的資料，留言板則是**很多人看同一份資料**——
你留言之後，別人打開頁面也看得到，這才是「網站」真正厲害的地方。

```
瀏覽器 (index.html)  →  伺服器 (app.py)  →  資料庫 (guestbook.db)
   每個人自己的畫面         所有人共用同一個         所有留言都存在這裡
```

## 檔案說明

| 檔案 | 角色 | 做什麼 |
|---|---|---|
| `templates/index.html` | 前端 | 留言表單＋清單，內容顯示前有做「逃逸處理」避免留言裡的符號搞壞畫面 |
| `app.py` | 後端 | 收留言、存資料庫、回傳全部留言（含時間） |
| `guestbook.db` | 資料庫 | SQLite 檔案，第一次執行時自動建立 |
| `run.bat` | 啟動器 | 雙擊啟動，網址 http://127.0.0.1:5051 |

## 怎麼驗證「真的是共用同一份資料」

自己在瀏覽器留言後，**换一個瀏覽器**（或用手機連同一個 Wi-Fi 開這台電腦的
IP:5051）打開同一個網址，會看到同樣的留言——因為資料存在伺服器的資料庫，
不是存在你自己瀏覽器裡。這跟記帳本每次只有你一個人在看，是不一樣的體驗。

## 這個範例故意留下的破洞（給你思考）

- **誰都能刪任何人的留言**：目前沒有「登入」機制，所以刪除鈕對所有人開放。
  真正的留言板會需要「這則是我發的，只有我能刪」——這就要碰到帳號系統了，
  是下一階段的功課，教學裡也提到「先跳過難的」，這裡先不做。
- **沒有防洗版機制**：同一個人可以無限狂發留言。真實網站通常會限制
  「幾秒內只能發一次」或要求登入。

## 延伸挑戰

1. **加已讀 / 按讚**：幫每則留言加一個讚數，練習「同一筆資料裡加一個可以累加的欄位」。
2. **分頁**：留言超過 50 則時，改成「一次只顯示 20 則，捲到底載入更多」。
3. **换成 MySQL**：跟記帳本一樣，把 SQLite 換成 MySQL 是很好的下一步練習。
